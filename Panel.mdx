# Pterodactyl Panel Documentation

## Overview

The **Pterodactyl Panel** is your central hub for managing game servers on XGamingServer. Key capabilities:

- **Real-time monitoring** of CPU, RAM, and player activity
- **One-click installations** for 30\+ supported games
- **Granular control** over files, databases, and user permissions

![Panel Dashboard Preview](/image_2025-03-29_193652370.png)

## Core Features

<Tabs>
  <Tab title="Tab">
    ### Server Controls

    Manage your game server's lifecycle and resources.

    #### Key Tabs:

    - **Console**: Execute commands and view live logs
    - **Startup**: Configure JVM flags or launch parameters
    - **Settings**: Allocate RAM/CPU or rename your server

    ```bash
    # Example: Restart server via CLI (for admins)  
    pterodactyl server restart --id=abc123
    ```

## User & Permission System

<Tabs>
  <Tab title="Tab">
    ### User Roles

    Invite team members with granular permissions:

    | Role          | Permissions                  |
    | ------------- | ---------------------------- |
    | **Admin**     | Full server control          |
    | **Moderator** | Restart server, kick players |
    | **Guest**     | View logs, no edits          |
  </Tab>
  <Tab title="Tab">
    ### Security Features

    - **Two-Factor Authentication (2FA)**: Enable in Account Settings
    - **API Access**: Generate tokens for automation
    - **Audit Logs**: Track user actions in **Activity** tab

    > ⚠️ **Best Practice**: Restrict admin access to trusted users only.
  </Tab>
</Tabs>

## Advanced Tools

### Network Configuration

- **Port Management**: Open/close ports for games or FTP
- **Firewall Rules**: Restrict IP ranges in **Network** tab

```json
// Example: Open Minecraft port (25565)  
{  
  "action": "allow",  
  "port": 25565,  
  "protocol": "tcp"  
}
```

## Troubleshooting

| Issue              | Solution                            |
| ------------------ | ----------------------------------- |
| Server won't start | Check **Console** logs for errors   |
| Permission denied  | Verify user roles in **Users** tab  |
| High CPU usage     | Reduce player slots or upgrade plan |
